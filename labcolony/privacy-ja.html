1. 端末内で取り扱う情報

クラウド同期を有効にしていない場合、本アプリの記録データは お使いの端末内のみ に保存されます。iOS アプリでは SQLite を利用します。 主なデータは以下のとおりです:

  • 部屋・ラック・ケージの情報 (名称、配置、状態、メモ等)
  • 個体情報 (個体コード、種別、系統、遺伝子型、性別、生年月日、ステータス等)
  • 個体のケージ所属履歴、移動履歴
  • 繁殖記録、死亡・安楽死記録
  • 投与記録、処置記録、体重記録
  • 個体識別情報 (耳パンチ、尾マーク、体表マーク、刺青等)
  • 定期作業、マスタ情報、種別ごとの設定、表示設定
  • バックアップ / 復元に必要なローカルデータ

これらの情報は、実験動物コロニーの記録・閲覧・集計・CSV 出力・バックアップ / 復元のために利用されます。 クラウド同期を有効にしない限り、開発者のサーバーへ送信されません。

入力する第三者情報について: 個体コード、系統、遺伝子型、処置内容、担当者名、メモ等に個人情報や機密情報を入力する場合は、 所属機関のルールに従い、管理に必要な最小限の情報にしてください。

2. クラウド同期で取り扱う情報

クラウド同期を有効にした場合、本アプリは同期対象の記録データを Supabase 上のクラウド環境へ送信します。 送信されるデータには、第 1 項に記載した部屋・ラック・ケージ・個体・繁殖・投与・処置・死亡・体重・ 定期作業・マスタ情報・設定等が含まれます。

クラウド同期は、同じアカウントの複数端末、または将来提供予定の研究室共有機能でデータを共有するために利用されます。 同期には Supabase Auth のアカウントが必要で、メールアドレス、認証トークン、ユーザー ID、 登録端末 ID、同期状態、エンタイトルメント情報等も扱います。

同期データは、ログイン中のユーザーまたは共有対象の研究室メンバーに限定して利用されるよう、 Supabase の Row Level Security、認証、同期 RPC による権限チェックを用いて管理します。 ただし、クラウド同期を有効にする場合は、ご自身の所属機関・研究室のデータ管理ルールに従ってください。

3. アカウント情報

購入の復元、クラウド同期、登録端末管理、研究室共有機能を利用する場合、アカウントへのログインが必要です。 本アプリは Supabase Auth を利用して、メールアドレス、認証トークン、ユーザー ID 等を扱います。 メールアドレスの変更やアカウント削除は、アプリ内の設定画面から行えます。

アカウント削除を実行すると、クラウド上のアカウント関連データと、この端末内のローカルデータを削除します。 通信エラー等でクラウド側の削除が完了しない場合は、オンラインで再度アカウント削除を実行してください。

4. 第三者サービスが取り扱う情報

4.1 Google AdMob (広告配信)

無料プランでは、Google AdMob を通じて広告が表示されます。 AdMob は広告配信のため、以下の情報を取り扱う場合があります:

  • 識別子 - 広告識別子 (IDFA): App Tracking Transparency (ATT) で許可いただいた場合のみ
  • 利用データ - 広告リクエスト、広告表示、広告 SDK 上のインタラクションに関する情報
  • 位置情報 - IP アドレスから推定される国・地域レベルのおおまかな位置情報
  • 診断情報 - クラッシュデータ、パフォーマンスデータ、SDK の技術的診断情報

本アプリは Google User Messaging Platform (UMP) による同意管理と、 iOS App Tracking Transparency (ATT) の許可状態に従って広告 SDK を利用します。 ATT を許可しない場合、IDFA は利用できません。対象地域では UMP の同意フォームが表示される場合があります。 有料プランでは広告は表示されません。

Google のプライバシーポリシー: https://policies.google.com/privacy
広告のパーソナライズ設定 / オプトアウト: https://adssettings.google.com/

4.2 Apple App Store / StoreKit

アプリ内課金の決済は Apple App Store / StoreKit を通じて処理されます。 開発者はクレジットカード番号などの決済情報にアクセスできません。 返金は Apple のポリシーに従って Apple が処理します。

Apple のプライバシーポリシー: https://www.apple.com/legal/privacy/

4.3 RevenueCat

本アプリは、アプリ内課金の状態確認、購入復元、サブスクリプションのエンタイトルメント管理のために RevenueCat を利用します。 RevenueCat は購入状態、アプリユーザー ID、製品 ID、サブスクリプション状態などを扱います。 本アプリでは、購入を正しいアカウントに紐づけるため、ログイン中の Supabase ユーザー ID を RevenueCat の app user ID として利用します。

RevenueCat のプライバシーポリシー: https://www.revenuecat.com/privacy/

4.4 Supabase

本アプリは、アカウント認証、クラウド同期、エンタイトルメント情報、登録端末管理、研究室共有機能のために Supabase を利用します。 クラウド同期を有効にした場合、同期対象データとアカウント関連情報が Supabase 上で処理されます。

Supabase のプライバシーポリシー: https://supabase.com/privacy

5. 本アプリ自体が収集しない情報

本アプリ自体は、以下の情報を目的外に収集しません (第 2 項のクラウド同期、第 4 項の第三者サービス、および第 6 項のお問い合わせメールによる任意提供分を除きます):

  • 端末の GPS 位置情報
  • 連絡先、写真、カレンダー、マイク
  • 医療・診療目的の患者情報
  • クレジットカード番号などの決済情報

6. メールでのお問い合わせ

legal@cedarlab.dev または support@cedarlab.dev 宛にお問い合わせいただいた場合、以下を取得・処理します:

7. 子どものプライバシー

本アプリは 13 歳未満を対象としていません。 13 歳未満のお子様の個人情報を意図的に収集することはありません。

8. データの保存と削除

端末内データは、アプリが端末に保存されている間保持されます。 アプリを削除することで、端末内の本アプリ関連データも削除されます。 アプリ内の操作で個別の記録を削除することもできます。

クラウド同期を有効にしている場合、クラウド上の同期データはアカウントおよび同期機能の提供に必要な範囲で保持されます。 アカウント削除により、クラウド上のアカウント関連データの削除を要求できます。

Google AdMob、Apple、RevenueCat、Supabase が保有する情報は各社のポリシーに従って管理されます。 各社のデータ削除・制限については、それぞれのプライバシーポリシーをご確認ください。

本アプリのデータ管理者 (controller) は日本在住の個人開発者 Akira Sugiyama (Cedarlab) です。 適用される範囲において、ご自身の個人データへのアクセス、訂正、消去、処理制限、異議申立て、 データポータビリティ、同意の撤回、監督機関への苦情申立て等の権利を有する場合があります。 これらの権利を行使される場合は、第 12 項のメールアドレスまでご連絡ください。
